## Task: Create UserController with CRUD endpoints

**What was done:**
Created UserController as a @RestController mapped to /users with all five CRUD endpoints:
- POST /users → 201 Created with Location header
- GET /users → 200 OK with list of users
- GET /users/{id} → 200 OK with single user
- PUT /users/{id} → 200 OK with updated user
- DELETE /users/{id} → 204 No Content
Wrote 5 unit tests using @WebMvcTest and MockMvc to test each endpoint in isolation.

**What was changed:**
- src/main/java/com/programacho/ralphloopexample/UserController.java (new)
- src/test/java/com/programacho/ralphloopexample/UserControllerTest.java (new)
- pom.xml (added spring-boot-webmvc-test dependency for @WebMvcTest)
- prd.md (marked task as complete)

**Remarks:**
In Spring Boot 4, @WebMvcTest moved from spring-boot-starter-test to a separate spring-boot-webmvc-test module, and the package changed from org.springframework.boot.test.autoconfigure.web.servlet to org.springframework.boot.webmvc.test.autoconfigure.

## Task: Add exception handling for 409 when the name or email is already taken

**What was done:**
Created UserAlreadyExistsException and added duplicate-check logic to UserService's createUser and updateUser methods.
Added existsByName and existsByEmail query methods to UserRepository.
Added a 409 Conflict handler in GlobalExceptionHandler.
The updateUser method only checks for conflicts when the name or email actually changes, allowing a user to be updated with its own existing values.
Wrote 7 new unit tests: 3 for UserService (create with duplicate name, create with duplicate email, update allowing same values) and 2 for UserService update conflicts, plus 2 controller tests for 409 on POST and PUT.

**What was changed:**
- src/main/java/com/programacho/ralphloopexample/UserAlreadyExistsException.java (new)
- src/main/java/com/programacho/ralphloopexample/UserRepository.java (modified — added existsByName, existsByEmail)
- src/main/java/com/programacho/ralphloopexample/UserService.java (modified — added duplicate checks in createUser and updateUser)
- src/main/java/com/programacho/ralphloopexample/GlobalExceptionHandler.java (modified — added 409 handler)
- src/test/java/com/programacho/ralphloopexample/UserServiceTest.java (modified — added 5 new tests, updated existing test stubs)
- src/test/java/com/programacho/ralphloopexample/UserControllerTest.java (modified — added 2 new tests for 409)
- prd.md (marked task as complete)

**Remarks:**
The updateUser duplicate check skips the existsBy query when the value hasn't changed, avoiding false conflicts when a user is saved with its own name/email. Total test count went from 22 to 31.

## Task: Add Spring Actuator and expose only the health endpoint

**What was done:**
Added spring-boot-starter-actuator dependency and configured management endpoints to expose only the health endpoint.
Wrote 2 integration tests: one verifying /actuator/health returns 200 with UP status, and one verifying /actuator/env returns 404 (not exposed).

**What was changed:**
- pom.xml (added spring-boot-starter-actuator, spring-boot-restclient, spring-boot-resttestclient)
- src/main/resources/application.yaml (added management.endpoints.web.exposure.include: health)
- src/test/java/com/programacho/ralphloopexample/ActuatorHealthTest.java (new)

**Remarks:**
In Spring Boot 4, TestRestTemplate moved from spring-boot-test to the spring-boot-resttestclient module (package org.springframework.boot.resttestclient), and it requires spring-boot-restclient as a transitive dependency for RestTemplateBuilder. Both were added as test-scoped dependencies. The @AutoConfigureTestRestTemplate annotation is needed alongside @SpringBootTest to auto-configure the TestRestTemplate bean. Total test count went from 31 to 33.
