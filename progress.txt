## Task: Create User entity

**What was done:**
Created the User JPA entity class with id (auto-generated Long), name (required, unique), and email (required, unique) fields.
Added Spring Web, Spring Data JPA, H2, and Spring Boot Test dependencies to pom.xml since the project had no dependencies.
Wrote unit tests verifying constructor and setter/getter behavior.

**What was changed:**
- pom.xml (added spring-boot-starter-web, spring-boot-starter-data-jpa, h2, spring-boot-starter-test)
- src/main/java/com/programacho/ralphloopexample/User.java (new)
- src/test/java/com/programacho/ralphloopexample/UserTest.java (new)

**Remarks:**
Used @Table(name = "users") to avoid conflict with the reserved SQL keyword "user".
Added a protected no-arg constructor for JPA and a public constructor taking name and email.

## Task: Create UserService with CRUD methods

**What was done:**
Created UserService with methods for all CRUD operations: createUser, findAllUsers, findUserById, updateUser, and deleteUser.
The service delegates to UserRepository and throws NoSuchElementException when an entity is not found during update or delete.
Wrote 8 unit tests using Mockito to mock UserRepository, covering all happy paths and error cases.

**What was changed:**
- src/main/java/com/programacho/ralphloopexample/UserService.java (new)
- src/test/java/com/programacho/ralphloopexample/UserServiceTest.java (new)
- prd.md (marked task as complete)

**Remarks:**
Used NoSuchElementException from Optional.orElseThrow() for not-found cases. This will be replaced with proper exception handling in a later task.

## Task: Create UserController with CRUD endpoints

**What was done:**
Created UserController as a @RestController mapped to /users with all five CRUD endpoints:
- POST /users → 201 Created with Location header
- GET /users → 200 OK with list of users
- GET /users/{id} → 200 OK with single user
- PUT /users/{id} → 200 OK with updated user
- DELETE /users/{id} → 204 No Content
Wrote 5 unit tests using @WebMvcTest and MockMvc to test each endpoint in isolation.

**What was changed:**
- src/main/java/com/programacho/ralphloopexample/UserController.java (new)
- src/test/java/com/programacho/ralphloopexample/UserControllerTest.java (new)
- pom.xml (added spring-boot-webmvc-test dependency for @WebMvcTest)
- prd.md (marked task as complete)

**Remarks:**
In Spring Boot 4, @WebMvcTest moved from spring-boot-starter-test to a separate spring-boot-webmvc-test module, and the package changed from org.springframework.boot.test.autoconfigure.web.servlet to org.springframework.boot.webmvc.test.autoconfigure.

## Task: Add exception handling for 404 when the user is not found

**What was done:**
Created a custom UserNotFoundException and a GlobalExceptionHandler (@RestControllerAdvice) that maps it to HTTP 404 with a JSON error body.
Updated UserService to throw UserNotFoundException (instead of NoSuchElementException) in findUserById, updateUser, and deleteUser.
Changed findUserById return type from Optional<User> to User since the exception is now thrown at the service layer.
Updated existing unit tests to expect UserNotFoundException and added 3 new controller tests for 404 responses on GET /users/{id}, PUT /users/{id}, and DELETE /users/{id}.

**What was changed:**
- src/main/java/com/programacho/ralphloopexample/UserNotFoundException.java (new)

- src/test/java/com/programacho/ralphloopexample/UserServiceTest.java (modified)
- src/test/java/com/programacho/ralphloopexample/UserControllerTest.java (modified)
- prd.md (marked task as complete)

**Remarks:**
Changed UserService.findUserById from returning Optional<User> to returning User directly, since the not-found case is now handled by throwing UserNotFoundException. This simplifies the controller code.
